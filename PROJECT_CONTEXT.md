# Project Context: Princessify

## 1. プロジェクト概要
**Princessify** は、ソーシャルゲーム『プリンセスコネクト！Re:Dive』の「クランバトル」を支援する多機能 Discord Bot です。主な機能は以下の4つです。

1. **お団子TL変換** (`@dango`): タイムラインのSET/UNSET状態を視覚化
2. **持ち越し計算** (`@mochi`): 同時凸の持ち越し時間を全組み合わせ探索
3. **凸ルート計算** (`@route`): 5パーティ×レンタル制約の3凸ルート全探索
4. **ランドソル杯** (`/landsol-cup`): ガチャ結果の集計・ランキング表示

## 2. ドメイン知識と用語定義
開発にあたり、以下のゲーム内仕様とユーザー文化を理解する必要があります。

* **UB（ユニオンバースト）:** キャラクターの必殺技。
* **SET（セット）機能:** UBゲージが溜まった瞬間に即発動するように予約するゲーム内機能。
* **オートON/OFF:** 全キャラの自動発動の切り替え。
* **お団子 / 旗揚げ:**
    * TLにおいて、特定のタイミングで「どのキャラをSET（予約）するか」「どのキャラのSETを解除するか」を視覚化したもの。
    * 5人のパーティメンバーの状態を `[〇〇ーーー]` のように表現する。
    * **出力の表記ルール:**
        * `〇`: SET状態を維持（KEEP ON）
        * `ー`: 非SET状態を維持（KEEP OFF）
        * `⭕`: 操作アクション（OFFからONにする＝SETする）
        * `❌`: 操作アクション（ONからOFFにする＝UNSETする）
    * **入力の揺らぎ対応:** ユーザー入力では様々な表記が使われるため、以下を認識して正規化する:
        * ON状態: `O`, `o`, `0`, `〇`, `◯`, `○`, `⭕`
        * OFF状態: `X`, `x`, `ー`, `❌`, `✕`, `✖`, `×`, `-`, `‐`, `−`, `–`, `—`
        * 括弧: `[]`, `［］`, `【】`, `()`, `（）`, `{}`, `｛｝`, `<>`, `＜＞`, `〈〉`, `《》`, `「」`, `『』`, `〔〕` → すべて `[]` に統一

## 3. 機能詳細

現在は **TypeScript (Node.js) + Discord.js** で実装されています。

### A. お団子TL変換 (`src/logic/princessify.ts`)

#### 既存モード（ユーザーが団子を記述）
1. **解析:**
    * 正規表現 `(\d{1,2}:\d{2})` で時間を検出。
    * 行内にパーティメンバーの名前が含まれているか検索（位置ベース：テキスト内の最初の出現位置で判定）。
    * 既存のお団子表記があれば解析し、状態配列（boolean[5]）に変換。
2. **ユーザー入力を正とする設計:**
    * ユーザーが記述したお団子状態を「正解」として扱う。
    * 最初の行は「初期状態」として、そのまま状態を表示（差分ではない）。
    * 2行目以降は、前の行との差分を計算して表示。
    * お団子がない行は、前の行の状態を継承する。
3. **描画と置換:**
    * 計算された状態変化を元に、`[〇⭕ーー❌]` のような文字列を生成。
    * 元のテキスト行に既存のお団子があれば置換し、なければ末尾に追加する。
    * ユーザーが書いたコメントは消さずに保持する。
4. **🌟目押しマーカー:** 手動目押しUBに🌟を付与（サブ行・空テキストは除外）。

#### 推論モード（お団子を自動生成）
`@dango`ヘッダー＋パーティ5人＋ボディにお団子なしの場合に発動。
1. **2パスアルゴリズム:**
    * Pass 1: SET ON/OFF、AUTO ON/OFF の需要（demand）を構築
    * Pass 2: 需要を適用して団子状態をレンダリング
2. **インライン命令対応:** `{名前}セット/解除`、`オートオン/オフ`、カンマ区切り名前展開
3. **初期状態行:** パーティ定義行〜最初のタイムスタンプ間の非空行を解析して初期状態に反映
4. **standalone on/off:** 裸の `on`/`off` をAUTO ON/OFFとして認識

#### ゲームコピー形式対応
* 「ユニオンバースト発動時間」行をTL開始マーカーとして検出（それ以前はプリアンブル）
* 複数行団子マージ: タイムスタンプ行の次行に団子/AUTO指示がある場合、前のエントリにマージ
* ○ (U+25CB) を ON として認識
* `AUTO入`/`AUTO入り`/`オート入` を AUTO ON として認識

#### channelMode
* `/dango add` で登録したチャンネルでは `@dango` 不要（最初の意味のある行がパーティ定義）
* タイムスタンプなしメッセージ → 無視（null）、タイムスタンプありパーティなし → PartyGuideError

### B. 持ち越し計算 (`src/logic/mochikoshi.ts`)

トリガー: `@mochi` / `@mo`

* 入力: `@mochi ボスHP ダメージ1 ダメージ2 ...`（最大10人）
* ラベル付き入力: `@mochi 500 Alice:3 Bob:2.8` や `3:Alice`
* 持ち越しマーカー: `*` or `💼` でキャラを持ち越し扱い（〆不可制約）
* `@mochi!` で万変換なし（raw値モード）
* 全サブセット（2人以上）×全〆パターンを探索し、有効な組み合わせをランキング表示
* ソート: 持ち越し秒数降順 → 参加人数降順 → 90秒同率時は〆ダメージ降順

### C. 凸ルート計算 (`src/logic/route.ts`)

トリガー: `@route` / `@rou` / `-route` / `-rou`

* 入力: 5行のパーティ構成（各5キャラ、`*` = レンタル必須）
* アルゴリズム: C(5,3)=10 組み合わせ × 5³=125 レンタルパターンの全探索
* 制約: (1) `*`付きキャラはレンタル必須 (2) 非レンタルキャラに重複なし
* 独立性判定: per-battle候補のデカルト積 == 有効パターン数なら独立 → `(X or Y)` マージ
* 非独立ルート: ≤5パターンならA/B列挙、>5なら `collapseRentalPatterns` でコンパクト表示
* ボス絵文字: `src/config/boss-emojis.json` から読み込み

### D. ランドソル杯 (`src/logic/landsol-cup.ts`)

スラッシュコマンド: `/landsol-cup`

* オプション: `mode` (top/bottom/all), `count` (表示件数), `date`, `since`, `detail`
* ガチャ結果パース: ハイフン/矢印/Unicode矢印区切りの数列(1-4)を部分抽出
* ジェムテーブル: {1:500, 2:400, 3:300, 4:250}
* ゲーム日境界: JST 5:00基準（UTC明示計算、ローカルTZ非依存）
* 日数不一致検出: 最頻値を基準、同率なら最大値
* フォーマット: メダル🥇🥈🥉（top）、💀🤡🫠（bottom）、全角スペースインデント
* since モード: 指定日以降を集計、各ユーザー最新結果使用、インライン注釈付き
* `date` と `since` は排他（同時指定不可）

## 4. 技術スタック
* **Runtime:** Node.js
* **Language:** TypeScript
* **Library:** discord.js, dotenv
* **Deploy:** Oracle Cloud Always Free VM (systemd service)
* **Environment:**
    * `.env` に `DISCORD_TOKEN` を保持。
    * `tsconfig.json` は `verbatimModuleSyntax: false`, `module: commonjs` 設定で動作中。
    * **重要:** サーバーはUTC → 時刻計算は必ずJST明示変換（ローカルTZ依存メソッド禁止）
    * **重要:** `tsc` はJSONを `dist/` にコピーしない → `process.cwd()` でパス解決

## 5. Discord制約への対応
* **2000文字制限:** `splitForDiscord(body, greeting)` で行単位分割・複数メッセージ送信
* **ページネーション:** ランドソル杯は `SnowflakeUtil` でチャンネル履歴を効率取得
